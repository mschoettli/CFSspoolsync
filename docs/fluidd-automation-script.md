# Fluidd Automation Script Guide

This guide documents the single automation entrypoint for patched Fluidd deployment.

Script file:
- `scripts/build_and_deploy_fluidd_cfs.ps1`

## Purpose

The script performs a full end-to-end deployment of patched Fluidd to the target host:
1. Clone Fluidd repository if missing.
2. Checkout the configured ref.
3. Apply CFS dashboard patch set automatically.
4. Build with `npm ci` and `npm run build`.
5. Optional fallback to prebuilt release artifact if clone/build fails and a fallback URL is provided.
6. Upload build files to the remote host.
7. Run remote deploy helper:
   - backup current `/usr/share/fluidd`
   - replace files
   - fix ownership and permissions
   - validate and reload nginx
8. Optional: write/update `:4409` reverse proxy in target nginx config and reload nginx.

## Prerequisites

- Windows PowerShell.
- Local tools in `PATH`:
  - `git.exe`
  - `npm.cmd`
  - `ssh.exe`
  - `scp.exe`
  - `tar.exe`
- SSH access to the Fluidd host (default user is `root`).
- Remote host must contain nginx and writable `/usr/share/fluidd`.

## SSH password prompts

If you use password-based SSH authentication (no SSH key), expect multiple prompts during one run.
Typical full run behavior is about 6 password prompts because the script performs several separate `ssh` and `scp` operations.

## Minimal command

Run from repository root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_and_deploy_fluidd_cfs.ps1 `
  -TargetHost "192.168.0.1" `
  -CfsHost "192.168.0.2" `
  -CfsPort 8088 `
  -ProxyPort 4409 `
  -ConfigureProxy `
  -UpdateRepo
```

## Parameters

- `-TargetHost` (required)
  - Hostname or IP of Fluidd nginx host.
- `-User` (default: `root`)
  - SSH user for remote commands.
- `-WorkspaceDir` (default: `$env:USERPROFILE\Documents\CodingProjects`)
  - Parent directory for auto-managed local checkout and fallback files.
- `-FluiddPath` (default: `<WorkspaceDir>\fluidd-cfs-auto`)
  - Local checkout path.
- `-RepoUrl` (default: `https://github.com/fluidd-core/fluidd.git`)
  - Fluidd repository URL.
- `-RepoRef` (default: `v1.36.4`)
  - Branch, tag, or commit to deploy.
- `-ApplyCfsPatch` (switch, default enabled)
  - Applies CFS widget/dashboard/layout patch set before build.
- `-PrebuiltAssetUrl` (default: empty)
  - Optional download URL used when repository clone/build path is unavailable.
  - If empty and build fails, the script stops with a clear error.
- `-UpdateRepo` (switch)
  - Runs `git fetch --all --tags --prune` before checkout.
- `-SkipBuild` (switch)
  - Skips local npm build and deploys existing `dist`.
- `-RemoteSourceDir` (default: `/tmp/fluidd-new`)
  - Temporary upload path on host.
- `-RemoteDeployScript` (default: `/tmp/deploy_fluidd_patch.sh`)
  - Remote path for deploy helper script.
- `-CfsHost` (optional, required when `-ConfigureProxy` is used)
  - Hostname or IP where CFS frontend is reachable.
- `-CfsPort` (default: `8088`)
  - CFS frontend port.
- `-ProxyPort` (default: `4409`)
  - Port exposed on Fluidd/nginx host for CFS iframe proxy.
- `-ConfigureProxy` (switch)
  - Adds or updates an autogenerated nginx server block:
    - `listen <ProxyPort>`
    - `proxy_pass http://<CfsHost>:<CfsPort>/`
    - websocket proxy on `/ws`
  - Reloads nginx after validation.

## Where to find the values

- `TargetHost`:
  - IP/host you use for Fluidd in browser (`http://<target-host>:4408`)
  - usually your K2/Fluidd nginx host
- `CfsHost`:
  - host where CFS frontend container is published
  - verify with `docker compose ps` in your stack directory
- `CfsPort`:
  - from your compose file (`compose.yaml` or `docker-compose.yml`) in `frontend.ports`
  - example `- "${HTTP_PORT:-8088}:80"` means external port `8088`
- `ProxyPort`:
  - nginx port on `TargetHost` used for iframe embed
  - default in this project is `4409`
- `User`:
  - SSH user for `TargetHost` (default `root`)

## Common variants

Update repository before checkout:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_and_deploy_fluidd_cfs.ps1 `
  -TargetHost "192.168.0.1" `
  -UpdateRepo
```

Fully automated setup (deploy + proxy config):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_and_deploy_fluidd_cfs.ps1 `
  -TargetHost "192.168.178.192" `
  -CfsHost "192.168.178.201" `
  -CfsPort 8088 `
  -ProxyPort 4409 `
  -ConfigureProxy `
  -UpdateRepo
```

Deploy existing `dist` without running npm build:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_and_deploy_fluidd_cfs.ps1 `
  -TargetHost "192.168.0.1" `
  -SkipBuild
```

## Validation

From Windows PowerShell:

```powershell
curl.exe -I "http://192.168.0.1:4408/"
curl.exe -I "http://192.168.0.1:4409/?view=fluidd&layout=card"
curl.exe -s "http://192.168.0.1:7125/server/extensions/list"
```

Expected:
- Fluidd root returns `200 OK`.
- `cfssync` appears in Moonraker extension agents (if agent is enabled).
